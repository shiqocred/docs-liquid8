## List Data

File yang berhasil di unggah akan disimpan di halaman <BU>Manifest Inbound</BU>
        
<ImageZoom src="/image/inbound/manifest/c-1.png" width={3276} height={2018} alt="Upload Inbound Process" className="rounded-lg m-0" />

### Deskripsi header tabel

| Nama          | Deskripsi                                          | Ket                                             |
|---------------|----------------------------------------------------|-------------------------------------------------|
| Document Name | Nama dari dokumen yang di unggah di Inbound Proses | -                                               |
| Date          | Tanggal file di unggah                             | -                                               |
| Total Items   | Total barang di dalam data                         | -                                               |
| Status        | Status Data                                        | [<Link className='size-4'/>](#list-status-data) |
| Action        | Opsi untuk data tersebut                           | [<Link className='size-4'/>](#list-action-data) |

### List status Data

| Nama          | Deskripsi                             |
|---------------|---------------------------------------|
| In Progress   | Data dalam proses penginputan barang  |
| Pending       | Data belum diproses atau data gagal   |
| Done          | Data telah selesai di proses          |

### List action Data

| Nama   | Icon                               | Deskripsi                             |
|--------|------------------------------------|---------------------------------------|
| Check  | <ShieldCheck className='size-4' /> | Melakukan input barang sesuai data    |
| Detail | <ReceiptText className='size-4' /> | Menampilkan halaman detail data       |
| Delete | <Trash2 className='size-4' />      | menghapus data                        |


## Detail Data

Terdiri dari **Metadata** dan **List Barang**, untuk memulai scan barang dapat menekan tombol **Next** untuk berpindah ke halaman **Check**
        
<ImageZoom src="/image/inbound/manifest/d-1.png" width={3276} height={2018} alt="Detail Inbound " className="rounded-lg m-0" />

**Barcode scan** merupakan kustom barcode untuk personalisasi barang dari sebuah data, misal barcode scan `3NAS` -> barcode baru barang `2400257` = `3NAS2400257`

<ImageZoom src="/image/inbound/manifest/e-1.png" width={400} height={239} alt="Barcode Product Inbound" className="rounded-lg m-0" />

**Detail scan** untuk menunjukan:
  - sudah berapa barang yang discan secara total dan sehari.
  - jumlah user yang men-scan data tersebut.
  - sudah berapa barang user men-scan barang dalam sehari.

<ImageZoom src="/image/inbound/manifest/f-1.png" width={2568} height={1870} alt="Barcode Product Inbound" className="rounded-lg m-0" />

### Deskripsi Metadata

| Nama       | Deskripsi    | Ket                                             |
|------------|--------------|-------------------------------------------------|
| Data Name  | Nama Data    | -                                               |
| Status     | Status Data  | [<Link className='size-4'/>](#list-status-data) |
| Merge Data | Kode data    | -                                               |
| Total      | Total barang | -                                               |


### Deskripsi header tabel

| Nama               | Deskripsi                      |
|--------------------|--------------------------------|
| No                 | Nomor urut barang didalam data |
| Barcode            | Barcode barang                 |
| Product Name       | Nama barang                    |
| Qty                | Kuantitas barang               |
| Price              | Harga barang dalam _satuan_    |
| Action: **Delete** | Hapus barang                   |

## Check Data

Halaman ini dapat di akses melalui tombol **Check** di halaman utama manifest inbound atau melalui tombol **Next** di detail manifest inbound.

<ImageZoom src="/image/inbound/manifest/g-1.png" width={2568} height={1870} alt="Check Product Inbound" className="rounded-lg m-0" />

Berikut Flow di halaman ini:

<Steps>
  <Step>
    Memasukan **Barcode Barang** di kolom pencarian
  </Step>
  
  <Step>
    Mengecek dan mempperbarui **data barang**
  </Step>
  
  <Step>
    Mengisi **kualitas barang**
  </Step>
</Steps>

### Pencarian Barcode Barang

Barcode barang akan otomatis dicari datanya setelah 0,5 detik tidak ada perubahan di kolom pencarian. Jenis barang dibagi menjadi 2 yaitu:

| Jenis    | Keterangan                      |
|----------|---------------------------------|
| Category | Harga Rp 100.000 atau diatasnya |
| Color    | Harga dibawah Rp 100.000        |

### Pengecekan dan pembaruan data

Data barang dibagi menjadi <BU>Old Data</BU> dan <BU>New Data</BU>

|---------|------------------|------------------------------|
